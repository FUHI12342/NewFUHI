## 1. エグゼクティブサマリー

**Repitte（リピッテ）** はLINE予約に特化した軽量SaaSで、月額2,200円からの低コストでLINE上の予約自動化を実現します。
一方 **NewFUHI（TimeBaiBai）** は、予約・POS・EC・IoT・HR・高度分析を一気通貫で提供するオールインワン店舗DXプラットフォームです。
**LINE予約だけなら Repitte、店舗経営のデジタル全体最適を目指すなら NewFUHI** が最適な選択です。

## 4. NewFUHI ならではの付加価値

### 4-1. オールインワンによる運用効率

```
Repitte利用時: 5〜6サービスを併用 → ログイン先が分散、データ連携なし、二重入力
NewFUHI利用時: 1つのダッシュボードで全て完結 → データ統合、自動分析、一元管理
```

- **データのサイロ化を解消**: 予約・売上・在庫・勤怠・来客データが1つのDBに集約
- **管理画面の一元化**: 複数サービスのログイン不要
- **自動連携**: 予約 → 来店 → 注文 → 決済 → 分析 → 施策提案の一気通貫フロー

### 4-2. AIによる経営支援（他社にない独自機能）

| 機能 | 効果 |
|------|------|
| AIインサイトエンジン | 売上急落・在庫危機・スタッフ過不足を自動検出し、具体的なアクションを提案 |
| 売上予測（Prophet） | 過去データから今後7〜30日の売上を予測、仕入れ・シフトの最適化 |
| メニューエンジニアリング | 利益率×人気度で商品を4象限分類、メニュー戦略を自動提案 |
| RFM分析 | 顧客を自動セグメント化し、VIP特典・再来店施策を推奨 |
| 需要予測ベース自動発注 | 来客予測×メニュー人気度×在庫から最適な発注量を算出 |

### 4-3. IoTによる安全管理（業界唯一）

- **CO（一酸化炭素）センサー**: シーシャ・焼肉・炭火等を扱う店舗の安全をリアルタイム監視
- **ガス警報 + 自動換気**: 閾値超過時にLINE緊急通知＋IRエアコン自動制御
- **人感センサー**: 来客数の自動カウント、混雑状況のリアルタイム把握

## 6. 導入ステップ（移行フロー）

### Phase 1: 準備・契約（1〜2週間）

```
Step 1: ヒアリング・要件確認
  └─ 現在のRepitte利用状況、他サービス併用状況を確認
  └─ 店舗の業態・規模・課題をヒアリング
  └─ 最適プラン（ライト/スタンダード/プレミアム）を提案

Step 2: デモ・トライアル
  └─ TimeBaiBai のデモ環境で全機能を体験
  └─ 自店舗のメニュー・スタッフでシミュレーション

Step 3: 契約・アカウント発行
  └─ 初期費用0円、月額プラン選択
  └─ 管理者アカウント発行
```

### Phase 2: データ移行・初期設定（1〜2週間）

```
Step 4: 基本設定
  └─ 店舗情報・営業時間・メニュー登録
  └─ スタッフアカウント作成
  └─ 決済設定（Coiney連携）

Step 5: データ移行
  └─ Repitteの顧客データ（CSV）をインポート
  └─ 予約履歴の移行（過去データの参照用）
  └─ LINE公式アカウントの連携設定

Step 6: IoTデバイス設置（プレミアムプランの場合）
  └─ Pico 2W デバイスキットの配送・設置
  └─ WiFi接続・センサーキャリブレーション
  └─ ダッシュボードでの動作確認
```

### Phase 3: 運用開始・最適化（2〜4週間）

```
Step 7: 並行運用期間
  └─ Repitte と NewFUHI を2週間並行運用
  └─ スタッフ向け操作研修（管理画面/POS/テーブルオーダー）
  └─ 顧客への新予約URLの案内

Step 8: 完全移行
  └─ Repitte の予約受付を停止
  └─ LINE公式アカウントのリッチメニューを NewFUHI に切替
  └─ Repitte 解約

Step 9: 運用最適化
  └─ 2週間後: 初回レビュー（利用状況・課題確認）
  └─ 1ヶ月後: AI分析データの蓄積確認、インサイト活用開始
  └─ 3ヶ月後: 定期レビュー、追加機能の活用提案
```
