2. エグゼクティブサマリ

NewFUHI は占いサロン・飲食店を対象とした統合管理プラットフォームです。以下の主要機能を提供します:

  • 予約管理: LINE OAuth / Email OTP による予約フロー、Coiney決済連携
  • テーブル注文: QRコードスキャンによるモバイルオーダー
  • ECショップ: オンライン商品販売（在庫連動）
  • IoTデバイス管理: Raspberry Pi Pico + センサーデータ収集、ガスアラート
  • 物件監視: IoTセンサーによる物件異常検知（ガス漏れ/長期不在/オフライン）
  • シフト・勤怠管理: シフト希望→自動スケジューリング→勤怠記録→給与計算
  • 在庫管理: 入庫QR / 出庫（注文連動） / 閾値アラート
  • セキュリティ監査: 12項目自己診断、ログ監視、AWSコスト監視
  • 多言語対応: 7言語（日本語/英語/繁体字/簡体字/韓国語/スペイン語/ポルトガル語）
  • CMS: ヒーローバナー、カスタムブロック、バナー広告
  • LINE連携: Webhookボット予約、リマインダー通知、セグメント配信、仮予約確認フロー
  • デモモード: 管理画面からON/OFF切替、デモデータの自動生成・表示制御
  • 自動バックアップ: SQLiteアトミックバックアップ、S3連携、保持ポリシー、管理画面設定

7.1 給与計算エンジン (payroll_calculator.py)

処理フロー

  1. 期間内の WorkAttendance 集計（通常/残業/深夜/休日の各分数）
  2. 総支給額計算: 基本給 + 残業手当(×1.25) + 深夜手当(×1.35) + 休日手当(×1.50) + 各種手当
  3. 社会保険料計算: 標準報酬月額 × 各料率（厚生年金/健康保険/雇用保険/介護保険/労災）
  4. 源泉徴収税: 課税対象額（総支給 - 社保 - 非課税通勤手当）から月額表参照
  5. 住民税: 固定月額加算
  6. PayrollEntry + PayrollDeduction レコード生成（update_or_create で冪等）

7.2 勤怠導出サービス (attendance_service.py)

  • 法定休憩: 6h以上→60分、4.5h以上→45分、それ以下→0分
  • 深夜帯: 22:00〜05:00
  • 法定労働: 1日8時間超は残業
  • 休日: 日曜日（weekday==6）

7.3 自動スケジューリング (shift_scheduler.py) — v1.1 カバレッジベース

  • ShiftStaffRequirement（定員）を時間帯ごとに適用
  • ShiftRequest(preferred) → 最優先割り当て（定員超過でも緩和判定）
  • ShiftRequest(available) → カバレッジ不足時のみ割り当て
  • ShiftRequest(unavailable) → 除外
  • 営業時間外リクエストのクリッピング（営業時間内に自動調整）
  • 最低連続勤務時間チェック（StoreScheduleConfig.min_shift_hours）
  • 不足枠(ShiftVacancy)自動生成: 定員未達の連続時間帯をマージ
  • Schedule同期: slot_durationに応じてShiftAssignment→Schedule分割

7.4 カバレッジ計算ヘルパー (shift_coverage.py) — v1.1 新規

  • build_coverage_map(): {date → {staff_type → {hour → [staff_ids]}}} マップ構築
  • record_assignment(): アサインをマップに記録
  • check_coverage_need(): リクエスト時間帯に定員未達の時間があるか判定
  • count_coverage_hours(): 不足時間数のカウント
  • generate_vacancies(): 不足枠レコード生成（連続時間マージ）
