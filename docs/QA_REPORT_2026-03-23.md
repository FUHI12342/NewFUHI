## 4. API エンドポイントテスト

### 正常応答 (認証不要)

| エンドポイント | ステータス |
|---------------|-----------|
| `GET /healthz` | 200 `ok` |
| `GET /api/currentTime` | 200 JSON |

### 認証必須 (403 = 正常動作)

テスト時: 7エンドポイントが正しく403を返却
修正後: 13エンドポイントが503→403に改善 (レートリミット緩和による)

### 修正前に503だったエンドポイント (全て修正済み)

`/api/dashboard/menu-engineering/`, `/api/dashboard/basket/`, `/api/dashboard/cohort/`,
`/api/dashboard/clv/`, `/api/dashboard/visitor-forecast/`, `/api/dashboard/sales-heatmap/`,
`/api/dashboard/aov-trend/`, `/api/dashboard/feedback/`, `/api/dashboard/nps/`,
`/api/menu`, `/api/analytics/visitors/`, `/api/shift/week-grid/`, `/api/pos/orders/`

**原因:** nginx rate limit `rate=30r/m` (毎分30リクエスト) が厳しすぎ、ダッシュボードの
並列API呼び出しでリミット超過していた。

**修正:** `rate=10r/s`, `burst=30` に緩和。
